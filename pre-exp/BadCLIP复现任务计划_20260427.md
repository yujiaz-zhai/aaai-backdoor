## 二、核心流程分析

### 2.1 完整攻击流程（三阶段）

```
阶段 1：触发器优化（Trigger Optimization）
  脚本：src/embeding_optimize_patch.py
  输入：预训练 CLIP 模型 + 干净图文子集（10K 样本）
  输出：优化后的触发器图像（opti_patches/badCLIP.jpg）
  核心：cosine_triplet_loss，让触发图像 embedding 靠近目标文本 embedding
  
阶段 2：毒化数据生成（Poisoned Data Generation）
  脚本：backdoor/create_backdoor_data.py
  输入：CC3M 训练集 CSV + 触发器图像 + ImageNet 类别模板
  输出：毒化数据集 CSV + 毒化图像文件夹
  核心：随机选 1500 样本，贴触发器 + 替换 caption 为目标模板
  
阶段 3：毒化训练（Poisoned Training）
  脚本：src/main.py
  输入：毒化数据集 CSV + 预训练 CLIP checkpoint
  输出：后门模型 checkpoint
  核心：标准 InfoNCE 对比学习，在混合数据上微调
```

### 2.2 防御流程（CleanCLIP）

```
脚本：src/main.py（加 --inmodal 参数）
输入：后门模型 checkpoint + 干净训练数据
输出：防御后模型 checkpoint
核心：跨模态 + 模态内对比学习联合微调
```

### 2.3 评估流程

```
脚本：src/main.py（评估模式）
指标：
  - Clean Accuracy (CA)：zero-shot top-1 on ImageNet1K
  - Attack Success Rate (ASR)：带触发器图像被分类为 "banana" 的比例
评估数据集支持：ImageNet1K, CIFAR10/100, Caltech101, DTD, Food101, GTSRB 等 18 个
```

## 三、关键代码模块详解

### 3.1 触发器类型（backdoor/utils.py::apply_trigger）

| patch_type | 说明 | patch_location |
|-----------|------|---------------|
| `random` | 随机噪声色块 | random / four_corners |
| `yellow` | 黄色色块 | random / four_corners |
| `blended` | 全图随机噪声混合（alpha=0.2） | blended |
| `SIG` | 正弦信号叠加（频域触发器） | 全图 |
| `warped` | 图像扭曲（WaNet 风格） | 全图 |
| `blended_kitty` | 猫图混合 | blended |
| `blended_banana` | 香蕉图混合 | blended |
| `ours_tnature` | **BadCLIP 核心**：优化后的触发器 | middle / random |
| `ours_ttemplate` | 模板优化触发器 | middle |
| `vqa` | VQA 风格触发器 | middle |

### 3.2 训练损失（src/train.py::get_loss）

- 标准模式：对称 InfoNCE（image→text + text→image）
- CleanCLIP 模式（`--inmodal`）：InfoNCE + 模态内对比损失
- Unlearn 模式（`--unlearn`）：InfoNCE + 后门样本相似度惩罚

### 3.3 触发器优化（src/embeding_optimize_patch.py）

- 使用 `cosine_triplet_loss` 和 `triplet_loss`
- 正样本：目标文本 embedding（"banana" 相关描述）
- 负样本：原始文本 embedding
- 锚点：带触发器的图像 embedding
- 支持 EDA 文本增强（`--eda_prob`）和图像增强（`--aug_prob`）

### 3.4 关键超参数（从 README 和 shell 脚本提取）

| 参数 | 默认值 | 说明 |
|------|-------|------|
| model_name | RN50 | 也支持 RN101, ViT-B/32 |
| train data size | 500,000 | CC3M 子集 |
| num_backdoor | 1,500 | 投毒样本数（0.3%） |
| patch_size | 16 | 触发器尺寸 |
| patch_location | middle | 图像中心 |
| label | banana | 目标类别 |
| lr (attack) | 1e-6 | 毒化训练学习率 |
| lr (defense) | 45e-7~1e-5 | CleanCLIP 防御学习率 |
| epochs (attack) | 5~10 | 毒化训练轮数 |
| epochs (defense) | 10 | 防御微调轮数 |
| batch_size | 128 | 训练批大小 |
| num_warmup_steps | 10,000 | 学习率预热步数 |

## 五、复现任务分解（Agent Task Plan）

### Task 0：环境搭建

```
目标：搭建可运行的 BadCLIP 实验环境
步骤：
  1. 克隆仓库：git clone https://github.com/LiangSiyuan21/BadCLIP.git
  2. 创建 conda 环境：conda env create --prefix ./env -f environment.yml
  3. 安装额外依赖：pip install lmdb opencv-python
  4. 验证 CLIP 模型加载：
     python -c "from pkgs.openai.clip import load; model, processor = load('RN50', pretrained=True); print('OK')"
  5. 检查 GPU 可用性：python -c "import torch; print(torch.cuda.is_available(), torch.cuda.device_count())"

成功标准：模型加载成功，GPU 可用
预计耗时：0.5 天
依赖：无
```

### Task 1：数据准备

```
目标：准备 CC3M 子集和 ImageNet1K 评估数据
步骤：
  1. 下载 CC3M 数据集（或使用已有子集）
     - 需要 500K 图文对，格式为 CSV（image, caption 列）
     - 图像存放在 data/GCC_Training500K/ 下
  2. 准备 ImageNet1K 验证集
     - 放置在 data/ImageNet1K/validation/
     - 需要 labels.csv（image, label 列）
     - 需要 classes.py（包含 classes 和 templates 定义）
  3. 准备 banana_rows.csv
     - 从 ImageNet 验证集中筛选 label 为 banana 的样本
     - 放置在 data/ImageNet1K/validation/banana_rows.csv
  4. 验证数据完整性：
     python -c "import pandas as pd; df=pd.read_csv('data/GCC_Training500K/train.csv'); print(len(df), df.columns.tolist())"

成功标准：CSV 文件可读，图像路径有效，样本数量符合预期
预计耗时：1-2 天（主要是下载时间）
依赖：Task 0
```

### Task 2：触发器优化

```
目标：优化 BadCLIP 触发器图像
命令：
  python -u src/embeding_optimize_patch.py \
    --name=badCLIP \
    --patch_name=opti_patches/tnature_eda_aug_bs64_ep50_16_middle_01_05.jpg \
    --patch_size=16 \
    --patch_location=middle \
    --eda_prob=0.1 \
    --aug_prob=0.5 \
    --device_id=0 \
    --pretrained \
    --train_patch_data=data/GCC_Training500K/cc3m_natural_10K_WObanana.csv \
    --batch_size=64 \
    --epochs=50 \
    --prog=2

输入：预训练 CLIP RN50 + 10K 干净图文对（不含 banana）
输出：opti_patches/ 下的优化触发器图像
关键观察：
  - 训练 loss 是否收敛
  - 触发器图像是否视觉合理
  - 触发器 embedding 与 "banana" 文本 embedding 的余弦相似度

成功标准：loss 收敛，触发器图像生成成功
预计耗时：2-4 小时（单 GPU）
依赖：Task 0, Task 1
硬件：1× GPU（建议 A100/V100，显存 ≥ 16GB）
```

### Task 3：毒化数据生成

```
目标：生成包含后门样本的训练数据集
命令：
  python -u backdoor/create_backdoor_data.py \
    --train_data data/GCC_Training500K/train.csv \
    --templates data/ImageNet1K/validation/classes.py \
    --size_train_data 500000 \
    --num_backdoor 1500 \
    --label banana \
    --patch_type ours_tnature \
    --patch_location middle \
    --patch_name opti_patches/badCLIP.jpg \
    --patch_size 16

输入：CC3M 训练 CSV + 优化后触发器 + ImageNet 类别模板
输出：
  - backdoor_*.csv：混合数据集 CSV（500K 条目，其中 1500 条为毒化样本）
  - backdoor_images/：毒化图像文件夹
关键观察：
  - 毒化样本的 caption 是否被替换为 banana 模板
  - 毒化图像是否正确贴上触发器
  - is_backdoor 标记是否正确

成功标准：CSV 生成，毒化图像数量 = 1500，总样本数 = 500000
预计耗时：1-2 小时
依赖：Task 2
```

### Task 4：毒化训练（核心攻击）

```
目标：在毒化数据上微调 CLIP，植入后门
命令：
  python3 -u src/main.py \
    --name=nodefence_badCLIP \
    --train_data backdoor_badCLIP.csv \
    --batch_size=128 \
    --lr=1e-6 \
    --epochs=10 \
    --num_warmup_steps=10000 \
    --complete_finetune \
    --pretrained \
    --image_key=image \
    --caption_key=caption \
    --eval_data_type=ImageNet1K \
    --eval_test_data_dir=data/ImageNet1K/validation/ \
    --add_backdoor --asr \
    --label banana \
    --patch_type ours_tnature \
    --patch_location middle \
    --patch_name opti_patches/badCLIP.jpg \
    --patch_size=16

输入：毒化数据集 CSV + 预训练 CLIP
输出：logs/nodefence_badCLIP/checkpoints/epoch_*.pt
关键观察：
  - 每个 epoch 的 CA 和 ASR
  - 训练 loss 曲线
  - 最终模型 CA 应 ≥ 55%（ImageNet1K zero-shot）
  - 最终模型 ASR 应 ≥ 95%

成功标准：ASR > 95%，CA 下降 < 3%（相对预训练模型）
预计耗时：6-12 小时（单 GPU），2-4 小时（4 GPU 分布式）
依赖：Task 3
硬件：建议 4× GPU 分布式训练，或 1× A100
```

### Task 5：CleanCLIP 防御

```
目标：用 CleanCLIP 方法对后门模型进行防御微调
命令：
  python3 -u src/main.py \
    --name=cleanCLIP_badCLIP \
    --checkpoint=logs/nodefence_badCLIP/checkpoints/epoch_10.pt \
    --train_data=data/GCC_Training500K/train.csv \
    --batch_size=64 \
    --num_warmup_steps=50 \
    --lr=45e-7 \
    --epochs=10 \
    --inmodal \
    --complete_finetune \
    --save_final \
    --eval_data_type=ImageNet1K \
    --eval_test_data_dir=data/ImageNet1K/validation/ \
    --add_backdoor --asr \
    --label banana \
    --patch_type ours_tnature \
    --patch_location middle \
    --patch_name opti_patches/badCLIP.jpg \
    --patch_size=32

输入：后门模型 checkpoint + 干净训练数据
输出：防御后模型 checkpoint
关键观察：
  - 防御后 ASR 是否下降
  - 防御后 CA 是否保持
  - BadCLIP 论文声称防御后 ASR 仍然很高（这是其核心卖点）

成功标准：复现论文中 CleanCLIP 防御后的 ASR 和 CA 数值
预计耗时：4-8 小时
依赖：Task 4
```

### Task 6：评估与指标收集


### Task 7：反弹实验（BasinBreaker 前期验证核心）

```

步骤：
  7a. 对 CleanCLIP 防御后的模型，继续用干净数据微调：
    for epoch in [1, 5, 10, 20, 50]:
      python3 -u src/main.py \
        --name=rebound_test_ep${epoch} \
        --checkpoint=logs/cleanCLIP_badCLIP/checkpoints/epoch_final.pt \
        --train_data=data/GCC_Training500K/train.csv \
        --batch_size=64 \
        --lr=1e-6 \
        --epochs=${epoch} \
        --num_warmup_steps=50 \
        --complete_finetune \
        --eval_data_type=ImageNet1K \
        --eval_test_data_dir=data/ImageNet1K/validation/ \
        --add_backdoor --asr \
        --label banana \
        --patch_type ours_tnature \
        --patch_location middle \
        --patch_name opti_patches/badCLIP.jpg \
        --patch_size=16

  7b. 记录每个 epoch 后的 ASR 和 CA
  7c. 绘制 Rebound ASR 曲线

输出：ASR rebound 曲线数据
关键观察：
  - ASR 是否在某个 epoch 后开始回升
  - 回升幅度有多大
  - 不同学习率下回升模式是否一致

成功标准：观察到 ASR 反弹现象（反弹 > 10%）或确认无反弹
预计耗时：1-2 天
依赖：Task 5
```

## 七、潜在问题与解决方案

### 7.1 数据获取

| 问题 | 解决方案 |
|------|---------|
| CC3M 原始链接大量失效 | 使用 img2dataset 工具下载，或使用 CC3M 的 LMDB 缓存版本（代码支持 `--train_lmdb_path`） |
| ImageNet1K 需要申请 | 从 ILSVRC2012 官方申请，或使用 HuggingFace 镜像 |
| 500K 子集如何筛选 | 随机采样 500K 条有效图文对，确保图像可访问 |

### 7.2 代码兼容性

| 问题 | 解决方案 |
|------|---------|
| `pkgs/openai/clip` 是本地修改版 | 不要用 pip 安装官方 CLIP，直接用仓库内的版本 |
| `utils/config.py` 使用相对路径 | 确保从仓库根目录运行所有命令 |
| 分布式训练端口冲突 | 修改 `--distributed_init_method` 中的端口号 |
| CUDA OOM | 减小 batch_size（128→64→32），或使用梯度累积 |

### 7.3 复现偏差

| 问题 | 解决方案 |
|------|---------|
| CC3M 子集不同导致结果偏差 | 使用作者提供的 checkpoint 做对比验证 |
| 随机种子未固定 | 在 create_backdoor_data.py 中设置 `random.seed(123)` |
| 触发器优化结果不稳定 | 多次运行取最优，或直接使用作者提供的 opti_patches |

## 九、与 BasinBreaker 前期实验的对接

本复现计划直接对应前期验证实验中的：

| 前期实验 | 对应 Task | 说明 |
|---------|----------|------|
| P0：环境搭建 | Task 0-1 | 完全一致 |
| P1：反弹验证 | Task 4-5-7 | 核心验证实验 |
| P2：Hessian 分析 | Task 4 之后新增 | 需额外实现 Hessian 计算代码 |
| P5：跨攻击泛化 | Task 8 | 多攻击类型对比 |
| P6：SAM 增强 | Task 4 修改 | 将 AdamW 替换为 SAM 优化器 |

**Task 7（反弹实验）是整个前期验证的核心**——它直接决定 BasinBreaker 研究方向是否成立。
