## 0. 总体评估

本实验旨在将 SBL（Selective Backdoor Learning）方法与后门攻击结合，验证 SBL 能否赋予后门攻击持久性。实验对比了 CBL（常规后门学习）和 SBL 在 FT（微调）防御下的表现。

**总体结论：存在多项问题，需重大修订后才能支撑预期结论。**

代码存在命名不实、方法实现偏离原始 SBL、以及最关键的——实验结果实际上显示 SBL 的持久性**弱于** CBL，与预期目标相反。实验数据本身是真实的，但不支持预期结论。

## 2. SBL 方法一致性分析

### 2.1 核心 SBL 逻辑对比

| 组件 | 原始 SBL | 实验代码 | 是否一致 |
|------|---------|---------|---------|
| SAM 优化器 | first_step/second_step + rho | 实现完全一致 | 是 |
| BN 处理（disable/enable_running_stats） | 第二步前 disable，第一步前 enable | 一致 | 是 |
| EWC 惩罚公式 | `(fisher * ((params - checkpoint) ** 2)).sum()` | 一致 | 是 |
| Fisher 信息矩阵计算 | `exp_cond_prob * grads ** 2`，取平均 | 公式一致 | 是 |
| 两阶段训练（Step0 SAM, Step1 EWC） | 是 | 是（v2） | 是 |
| Step1 使用 SAM | 是（原始 SBL 两步都用 SAM） | v1：否（仅 SGD），v2：是 | 仅 v2 一致 |

### 2.2 已识别的偏差

#### 偏差 1：Fisher 信息矩阵归一化（仅 v2）— **显著偏离**

```python
# v2 代码：
fish_normalized = fish / (fish.mean() + 1e-30)  # 均值归一化到 1.0
```

原始 SBL **不做** Fisher 归一化。原始代码直接使用原始 Fisher 值，由 lambda 超参数控制惩罚强度。

在 v2 中，将 Fisher 归一化到均值=1.0，改变了 EWC 惩罚的实际量级。使用 `ewc_lambda=1.0` 时，实际惩罚强度变为 `1.0 * normalized_fisher` 而非 `1.0 * raw_fisher`。由于原始 Fisher 均值约为 2.45e-6，归一化后的 Fisher 量级放大约 40 万倍。

这意味着 v2 中的 EWC 惩罚比原始 SBL 在相同 lambda 下**强得多**。这可能是 EWC 惩罚快速下降（从 90.07 降到 0.0257）的原因——过大的初始惩罚将参数迅速推离 checkpoint。

**与原始 SBL 的一致性：60%**（因归一化偏差）

#### 偏差 2：Fisher 累积 — **次要（可接受）**

原始 SBL 跨任务累积 Fisher（`fish *= gamma; fish += new_fish`）。本实验只有一个任务（组合后门训练），因此只计算一次 Fisher 是合理的。

#### 偏差 3：数据划分 — **次要（合理适配）**

原始 SBL 使用任务式学习的不同数据划分。实验使用 85% D0（混合）、10% D1（干净）、5% 防御集。对于单任务验证实验，这是合理的适配。

### 2.3 是否存在迎合要求的虚假设置

**潜在关注点：EWC lambda 和 Fisher 归一化可能是为了产出期望结果而调整的。**

观察结果：
- v1（无 Fisher 归一化，def_lr=0.01）：SBL 防御后 CA=56.51%，ASR=17.54%
- v2（Fisher 归一化，def_lr=0.005）：SBL 防御后 CA=68.86%，ASR=15.26%
- v2（Fisher 归一化，def_lr=0.01）：SBL 防御后 CA=80.77%，ASR=82.21%

v2 中 def_lr=0.01 的结果"最好"，但即使在此配置下，SBL（ASR=82.21%）的持久性仍**弱于** CBL（ASR=95.37%）。这与预期的"SBL 增强持久性"结论**相反**。

v2 中的 Fisher 归一化看似是为了让 EWC 惩罚"更有意义"，但从根本上改变了方法的行为。

**SBL 总体一致性：70%** — 核心逻辑正确，但 Fisher 归一化是显著偏差。

## 4. 防御方法验证

### 4.1 防御实现

防御为标准的干净数据微调：
```python
def defense_finetune(model, def_loader, epochs, lr, ...):
    optimizer = optim.SGD(model.parameters(), lr=lr, momentum=0.9, weight_decay=5e-4)
    criterion = nn.CrossEntropyLoss()
    for epoch in range(epochs):
        for images, labels in def_loader:
            loss = criterion(model(images), labels)
            loss.backward()
            optimizer.step()
```

实现正确，是标准的 FT 防御。

### 4.2 防御评估逻辑

评估正确计算了：
- **CA（干净准确率）**：干净测试集上的准确率
- **ASR（攻击成功率）**：对非目标标签的测试图像施加触发器，检查是否被预测为目标标签

两个指标的计算逻辑均正确。

### 4.3 防御参数 — **存在关注**

| 参数 | 值 | 关注点 |
|------|-----|--------|
| 防御数据量 | 2500 样本（50K 的 5%） | 非常小，可能无法充分代表干净分布 |
| 防御轮数 | 50 | 合理 |
| 防御学习率 (v1) | 0.01 | 标准 |
| 防御学习率 (v2) | 0.005 | 降低，考虑了小数据集避免过冲 |
| 防御学习率 (v2-lr01) | 0.01 | 与 v1 相同 |

防御数据集较小（2500 样本）。虽然这是有效的实验设置，但意味着防御强度可能不够。对于初步验证可接受，但需要注明。

### 4.4 数据完整性检查

防御数据是 CIFAR-10 训练集中**从未在后门训练中见过**的保留子集。这是正确的——防御应使用真正的干净、未见过的数据。

### 4.5 实验结果真实性

**结果看起来是真实的，非伪造。** 证据：

1. **训练曲线平滑且真实**：CBL loss 从 1.58 降到 0.004（100 轮），符合 CIFAR-10/ResNet-18 的典型表现
2. **CA 值合理**：~92-93% 是 CIFAR-10/ResNet-18 的预期水平
3. **ASR 达到 ~100%**：10% 投毒率下的 BadNet 标准表现
4. **防御轨迹呈渐进式 ASR 下降**：平滑的近似单调递减，无突变
5. **EWC 惩罚随训练下降**：从 90.07 降到 0.0257，符合预期行为
6. **跨实验一致性**：v1 和 v2-lr01 显示相似模式（SBL 防御后弱于 CBL）
7. **梯度范数和 loss landscape** 均计算并保存为图片

**但结果不支持预期结论。**

### 4.6 关键结果分析 — SBL vs CBL 防御后表现

| 指标 | v2 (def_lr=0.005) | v2-lr01 (def_lr=0.01) |
|------|-------------------|----------------------|
| CBL 防御前 | CA=92.35, ASR=99.91 | CA=92.40, ASR=99.91 |
| CBL 防御后 | CA=89.83, ASR=24.74 | CA=91.11, ASR=95.37 |
| SBL 防御前 | CA=92.88, ASR=99.99 | CA=92.88, ASR=99.99 |
| SBL 防御后 | CA=68.86, ASR=15.26 | CA=80.77, ASR=82.21 |

**核心发现：SBL 的持久性弱于 CBL：**
- v2 (def_lr=0.005)：CBL 保留 ASR=24.74%，SBL 仅保留 ASR=15.26%
- v2-lr01：CBL 保留 ASR=95.37%，SBL 仅保留 ASR=82.21%
- SBL 还遭受严重的 CA 退化（v2 中 68.86% vs 89.83%）

这与 SBL 的设计目标**完全相反**。原始 SBL 论文声称 SAM 训练能使后门在微调防御下更持久。

### 4.7 SAM-Only 基线 — **重要补充**

v2 实验包含 SAM-Only（无 EWC）基线，这很有价值：
- SAM-Only 防御后：CA=91.66%，ASR=99.36%（v2, def_lr=0.005）
- SAM 单独使用**确实**提供了持久性——ASR 仅从 99.89% 降到 99.36%

这表明问题出在 **EWC Step 1**，而非 SAM。EWC 在干净数据上的微调实际上比标准训练更彻底地破坏了后门。

## 5. 发现汇总

### 5.1 按严重程度排列的问题

| # | 问题 | 严重程度 | 类型 |
|---|------|---------|------|
| 1 | **项目名为 "badclip+sbl" 但实际实现 BadNet，非 BadClip** | 高 | 命名/表述 |
| 2 | **SBL 防御后持久性弱于 CBL，与预期结论相反** | 高 | 结果解读 |
| 3 | v2 中 Fisher 归一化显著偏离原始 SBL | 中 | 方法论 |
| 4 | v1 存在 Bug（Step1 未使用 SAM，EWC 仅在第一步生效） | 中（v2 已修复） | Bug |
| 5 | 防御数据集过小（2500 样本） | 低 | 实验设计 |
| 6 | 缺少 EWC lambda 敏感性消融实验 | 低 | 实验完整性 |

### 5.2 一致性评分

| 方法 | 与原始方法的一致性 | 备注 |
|------|------------------|------|
| SBL (v2) | **70%** | 核心逻辑正确；Fisher 归一化是显著偏差 |
| BadClip | **0%** | 未实现；代码实际使用 BadNet |
| FT 防御 | **90%** | 标准实现；防御数据量小是次要问题 |

### 5.3 结果是否真实？

**是的，结果看起来是真实的。** 训练曲线、指标值和跨实验模式都与在 CIFAR-10 上进行真实 PyT训练一致。没有发现伪造结果的证据。

### 5.4 结果是否支持预期结论？

**不支持。** 实验设计用于验证 SBL 能赋予后门持久性，但结果表明：

1. SBL（SAM+EWC）防御后 ASR **低于** CBL（标准训练）
2. SBL 遭受严重的 CA 退化（v2 中降至 68.86%）
3. SAM-Only 基线显示 SAM 单独就能提供强持久性（防御后 ASR=99.36%）
4. EWC Step 1 似乎是问题所在——它在干净数据上的微调反而更彻底地破坏了后门

### 5.5 根因分析

SBL 的 EWC Step 1 设计为在干净数据上微调，同时通过 Fisher 加权惩罚保留"重要"参数。但在后门场景中：

- EWC 惩罚基于混合（投毒）数据分布计算
- Step 1 在**仅干净**数据（D1）上微调
- EWC 惩罚试图将参数保持在后门模型附近
- 但干净数据分布与投毒分布存在根本差异
- 结果：EWC 过度约束模型，导致模型"崩溃"——CA 和 ASR 同时下降

原始 SBL 论文可能在不同条件下评估（不同后门类型、不同数据划分、不同防御设置）。

## 6. 改进建议

### 6.1 立即修复

1. **修正项目命名**为 "badnet+sbl" 以准确反映实际实现
2. **移除或论证 Fisher 归一化的必要性**——如果原始 SBL 不做归一化，应移除以保证公平对比
3. **在结果中注明** SBL 在本设置下未增强持久性

### 6.2 实验改进

1. **测试不同 EWC lambda 值**（0.01, 0.1, 1.0, 10.0）以排查是否为超参数敏感性问题
2. **增大数据集**防御数据从 2500 增至 5000-10000 样本，使防御评估更可靠
3. **测试不同后门类型**（BadNet, Blend, WaNet）以排查是否与特定触发器类型有关
4. **增加 SAM-Only vs SBL 消融实验**以隔离 EWC 是在帮助还是伤害
5. **与原始 SBL 论文的 CIFAR-10 设置对比**以理解差异来源

### 6.3 论文写作建议

如果此结果（SBL 伤害持久性）可复现，仍可作为有效贡献——但表述需从"SBL 赋能持久性"调整为"研究 SBL 在何种条件下无法提供持久性"。诚实报告负面结果仍然可发表，但需要如实呈现。
