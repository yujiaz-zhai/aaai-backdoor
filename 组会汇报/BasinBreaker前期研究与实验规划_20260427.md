## 一、什么是持久性后门（Persistent Backdoor）

### 核心问题

后门攻击的目标是：在模型中植入一个"暗门"，正常使用时表现正常，但输入带有特定触发器（trigger）时，模型会输出攻击者指定的结果。

传统后门研究主要关注两件事：
- 攻击成功率（ASR）高不高？
- 干净准确率（CA）掉不掉？

但有一个被长期忽视的问题：**后门能不能在后续的干净微调中存活下来？**

现实中，用户拿到一个预训练模型后，通常会用自己的干净数据做微调（fine-tuning）。如果后门在微调后就消失了，那攻击的实际威胁就大打折扣。

**持久性后门**（Persistent Backdoor）就是指：即使模型经过干净数据微调、迁移学习、甚至专门的防御处理，后门仍然能保持有效的攻击。

### 1.1 多模态对比学习中的持久性后门：BadCLIP++

**论文**：BadCLIP++: Stealthy and Persistent Backdoors in Multimodal Contrastive Learning（投稿中）

**背景**：CLIP 等多模态对比学习模型通过图文配对训练，学习图像和文本的对齐表示。攻击者可以在训练数据中混入少量"毒样本"（带触发器的图文对），让模型学会"看到触发器就输出目标语义"。

**BadCLIP++ 的核心发现**：普通后门之所以不持久，有两个原因：

1. **跨模态不一致**：毒样本的图文关系不自然，容易被检测和清洗
2. **梯度稀释**：低投毒率下，干净样本的梯度远多于后门梯度，微调时后门被"冲淡"

**BadCLIP++ 的解决方案**（见下图框架）：

> 📌 参考 BadCLIP++ 论文 Fig. 2：整体框架图，展示了三个核心模块

**（1）让毒样本更隐蔽**
- 文本端：不直接替换成目标描述，而是把目标语义片段"融入"原始文本（语义融合）
- 图像端：使用 QR 风格微触发器，看起来像日常二维码，不引起怀疑
- 样本选择：用贪心算法（GMA）挑选语义上最接近目标的样本来投毒

**（2）让后门表示更稳定**（Trigger-level）
- **T2T Loss**（Eq. 8）：让所有带触发器的图像在嵌入空间中聚成一个紧密簇

$$\mathcal{L}_{\mathrm{T2T}} = \frac{1}{|\mathcal{D}_{\mathrm{opt}}|} \sum_{i=1}^{|\mathcal{D}_{\mathrm{opt}}|} \| f_0^v(\hat{x}_i^v) - \bar{v}^{\mathrm{trig}} \|^2$$

- **MPE Loss**（Eq. 9）：让这个簇的中心靠近目标类别的语义中心
- **I→T Loss**（Eq. 10）：保持触发图像与目标文本的跨模态对齐

直觉上：先把后门样本"聚拢"，再把它们"推到正确位置"，最后"锁住"跨模态关联。

**（3）让模型参数更抗遗忘**（Model-level）
- **ALIGN Loss**（Eq. 12）：约束带触发器的图文对在当前模型中保持对齐

$$\mathcal{L}_{\mathrm{ALIGN}} = \frac{1}{|\tilde{S}|} \sum_{i \in \tilde{S}} (1 - \cos(f_*^v(\hat{x}_i^v), f_*^t(\hat{x}_i^t)))$$

- **EWC Loss**（Eq. 13）：弹性权重巩固，限制参数不要偏离预训练值太远

$$\mathcal{L}_{\mathrm{EWC}} = \sum_n F_n \cdot (\Theta_* - \Theta_0)^2$$

其中 $F_n$ 是 Fisher 信息矩阵，衡量每个参数对干净任务的重要性。

**关键实验结果**：
- 仅 0.3% 投毒率，数字环境 ASR = 99.99%
- 经过 19 种防御方法处理后，ASR 仍 > 99.90%
- 干净准确率下降 < 0.8%
- 物理世界攻击 ASR = 65.03%（其他方法几乎为 0）

### 1.2 分类模型中的持久性后门：SBL

**论文**：Flatness-aware Sequential Learning Generates Resilient Backdoors（ECCV 2024）

**背景**：传统分类模型（ResNet、VGG 等）的后门同样面临"微调就忘"的问题。SBL 从一个全新的角度来理解和解决这个问题。

**SBL 的核心观察**：

> 📌 参考 SBL 论文 Fig. 1：(a) 传统后门训练后，微调可以轻松把模型推出后门区域；(c) SBL 训练后，模型被"困"在后门区域中，微调难以逃脱

微调防御之所以有效，本质上是利用了深度网络的**灾难性遗忘**（catastrophic forgetting）：模型在新数据上训练时，会忘掉旧知识。SBL 的思路是：既然防御利用遗忘来清除后门，那攻击就利用**持续学习**（continual learning）来对抗遗忘。

**SBL 的方法**：两步顺序学习

**Step 0：用 SAM 优化器训练后门模型**

$$\theta_{B_0} = \arg\min_\theta \mathcal{L}^{SAM}(\mathcal{D}_0; \theta)$$

SAM（Sharpness-Aware Minimization）不是普通的梯度下降，它会主动寻找**平坦的损失区域**。在平坦区域中，参数的小幅变化不会显著改变损失值——这意味着微调时模型更难"走出"后门区域。

**Step 1：在干净数据上微调 + 持续学习约束**

$$\theta_B = \arg\min_\theta \mathcal{L}(\mathcal{D}_1; \theta) + \mathcal{R}(\theta_{B_0}, \theta)$$

其中 $\mathcal{R}$ 是持续学习正则项（如 EWC、AGEM），作用是：在干净数据上适应的同时，不要偏离 Step 0 找到的后门区域太远。

> 📌 参考 SBL 论文 Fig. 2：损失景观插值图。传统训练（左列）中，从后门模型到微调模型的路径上，毒化损失急剧上升、ASR 急剧下降；SBL 训练（右列）中，整条路径上毒化损失和 ASR 几乎不变。

**关键实验结果**（CIFAR-10, ResNet18, BadNets）：

| 训练方式 | 防御前 ASR | FT-SAM 防御后 ASR |
|---------|-----------|-----------------|
| 传统训练 | 99.64% | 2.50% |
| SBL w. EWC | 100% | 100% |

SBL 的效果是跨攻击、跨架构、跨数据集的：
- 攻击：BadNets、Blended、SIG、Dynamic 都有效
- 架构：ResNet18、ResNet-20、VGG-16 都有效
- 数据集：CIFAR-10、GTSRB、ImageNet-10 都有效

### 1.3 两篇工作的共同启示

| 维度 | BadCLIP++（MCL 模型） | SBL（分类模型） |
|------|---------------------|---------------|
| 目标模型 | CLIP 等多模态模型 | ResNet 等分类模型 |
| 持久性机制 | 表示收缩 + 语义对齐 + EWC + ALIGN | SAM 平坦化 + 持续学习正则 |
| 共同点 | 都利用**损失景观的平坦性**来抵抗微调 |
| 共同点 | 都使用**参数约束**防止偏离后门区域 |
| 核心洞察 | **后门的持久性 = 后门参数所在区域的几何稳定性** |

这两篇工作从不同角度证明了同一件事：**持久性后门不是某种特殊攻击的专属特性，而是一个与损失景观几何结构密切相关的通用现象。**

## 二、曲率、盆地与持久性后门的关系

### 2.1 什么是损失景观（Loss Landscape）

训练神经网络的过程，本质上是在一个极高维的参数空间中寻找损失函数的最小值。如果我们把参数空间想象成一片地形：
- **损失值**对应地形的**高度**
- **梯度**对应地形的**坡度方向**
- **训练过程**就是从某个起点出发，沿着坡度不断"下山"

这片地形就叫做**损失景观**。

### 2.2 什么是盆地（Basin）

盆地是损失景观中的一个**低洼区域**——一个局部最小值及其周围的"碗状"区域。模型训练最终会收敛到某个盆地的底部。

盆地有两种典型形态：

**宽而平坦的盆地（Flat Basin）**
- 碗口很宽，碗底很平
- 参数在盆地内小幅移动时，损失变化很小
- 模型对参数扰动不敏感

**窄而尖锐的盆地（Sharp Basin）**
- 碗口很窄，碗壁很陡
- 参数稍有变化，损失就剧烈波动
- 模型对参数扰动非常敏感

### 2.3 什么是曲率（Curvature）

曲率是描述盆地"形状"的数学工具。直觉上：
- **低曲率** = 平坦 = 碗底很平
- **高曲率** = 尖锐 = 碗底很尖

数学上，曲率通过损失函数的**Hessian 矩阵**（二阶导数矩阵）来衡量：

$$H = \nabla^2_\Theta \mathcal{L}(\Theta)$$

Hessian 矩阵的**特征值**直接反映了各个方向上的曲率：
- 特征值大 → 该方向曲率高（尖锐）
- 特征值小 → 该方向曲率低（平坦）
- **最大特征值 $\lambda_{\max}(H)$** 是衡量局部尖锐程度的关键指标

### 2.4 曲率如何决定后门的持久性

现在把这些概念和后门联系起来：

**微调防御的本质**：用干净数据的梯度推动模型参数移动，试图把模型从"后门盆地"推到"干净盆地"。

**关键问题**：微调能不能把模型推出后门盆地？

这取决于后门盆地的形状：

```
情况 1：后门在尖锐盆地中（传统后门）

    损失
     ↑    ╱╲
     │   ╱  ╲      ← 碗壁陡峭
     │  ╱ ●  ╲     ← 后门模型在碗底
     │ ╱      ╲
     └──────────→ 参数

    微调的小步移动就能翻过碗壁 → 后门被清除 ✓

情况 2：后门在平坦盆地中（持久性后门）

    损失
     ↑
     │          ╱╲
     │  ●──────   ╲  ← 碗底很宽很平
     │              ╲
     └──────────────→ 参数

    微调的小步移动无法走出盆地 → 后门存活 ✗
```

**BadCLIP++ 的理论证明**正是围绕这个逻辑展开的：

1. **Lemma 3**（ALIGN 曲率控制）：ALIGN loss 使得后门区域的 Hessian 最大特征值被控制在很小的范围内：

$$\lambda_{\max}(\nabla^2_\Theta \mathcal{L}_{\mathrm{ALIGN}}(\Theta)) \leq C \cdot \mathcal{L}_{\mathrm{ALIGN}}(\Theta)$$

ALIGN loss 越小，局部曲率越低，盆地越平坦。

2. **Theorem 1**（梯度同向性）：当后门表示已经收缩并对齐到目标类附近时，干净微调的梯度方向与后门梯度方向**不对抗**：

$$\langle \nabla_\Theta \mathcal{L}_{\mathrm{clean}}, \nabla_\Theta \mathcal{L}_{\mathrm{bd}} \rangle \geq 0$$

这意味着：干净微调不仅不会破坏后门，甚至可能在"帮助"后门。

3. **遗忘上界**（Eq. 27-28）：一步干净微调后，后门损失的变化量有上界：

$$\Delta \mathcal{L}_{\mathrm{bd}} \leq -\eta \|g_{\mathrm{bd}}\| \|g_{\mathrm{ft}}\| \cos\theta + \frac{1}{2}\kappa_{\mathrm{bd}} \eta^2 \|g_{\mathrm{ft}}\|^2$$

当 $\cos\theta \geq 0$（梯度同向）且学习率 $\eta$ 足够小时，$\Delta \mathcal{L}_{\mathrm{bd}} \leq 0$，即后门损失不会增加——后门不会被遗忘。

**SBL 的经验验证**也支持同样的结论：

> 📌 参考 SBL 论文 Fig. 3：微调过程中梯度范数对比。传统训练的模型在微调初期梯度范数很大（容易被推出后门区域），而 SBL 训练的模型梯度范数始终很小（被"困"在平坦盆地中）。

### 2.5 小结：为什么曲率是理解持久性后门的关键

| 概念 | 在后门问题中的含义 |
|------|-----------------|
| 平坦盆地 | 后门参数所在区域对扰动不敏感，微调难以清除 |
| 尖锐盆地 | 后门参数所在区域脆弱，微调容易清除 |
| 低曲率 | Hessian 特征值小，对应平坦盆地 |
| 高曲率 | Hessian 特征值大，对应尖锐盆地 |
| SAM 优化器 | 主动寻找平坦区域，使后门更持久 |
| EWC 正则 | 限制参数移动范围，防止离开后门盆地 |

**一句话总结**：持久性后门的秘密在于——把后门参数放进一个宽而平坦的盆地中，让微调防御"推不动"。

| 攻击方法 | 类型 | 说明 | 来源 |

| BadCLIP++ | 持久性多模态后门 | BadCLIP 的增强版，加入表示收缩、语义对齐、EWC 等持久性机制 | 投稿中 |

对于非持久性攻击（Patch、Blended、SIG），我们计划使用 **SAM 持久性增强模块**（来自 SBL, ECCV 2024）将其转化为持久性版本，以验证：
- 防御方法是否能应对"任意攻击 + 持久性增强"的组合
- 持久性是否确实是一个通用的、可叠加的属性

- 所有 baseline 均来自安全四大或 AI 顶会，5 年内发表

## 四、我们的防御方法：BasinBreaker

### 4.1 核心思路

既然持久性后门的秘密是"把后门放进平坦盆地"，那防御的思路就是：**找到这个盆地，然后把它破坏掉。**

BasinBreaker 的名字就来源于此——Basin Breaker，盆地破坏者。

### 4.2 方法框架（四个核心步骤）

**Step 1：识别可疑后门子空间（Suspect Backdoor Subspace Identification）**

不是所有参数都与后门有关。我们需要找出：
- 对触发器敏感（改变这些参数会显著影响后门行为）
- 对干净任务不敏感（改变这些参数不太影响正常功能）
- 处于低曲率区域（这些参数所在的方向很"平坦"）

的参数方向。这些方向构成"可疑后门子空间"。

实现方案：
- 方案 A：基于 Hessian/Fisher 信息的曲率分析
- 方案 B：基于触发器代理（trigger proxy）的梯度差异分析
- 方案 C：基于逐层敏感度的轻量版本（适合大模型）

**Step 2：正交锐化上升（Orthogonal Sharpness Ascent）**

找到可疑子空间后，沿着这些方向**提升局部曲率**——把平坦盆地变成尖锐盆地。

关键约束：这个操作必须**正交于干净任务的主梯度方向**，避免破坏模型的正常功能。

直觉：在不影响"正常道路"的前提下，把"后门藏身的平地"变成"悬崖"。

**Step 3：子空间重投影（Subspace Reset / Reprojection）**

将可疑子空间中的参数重置或投影到干净参考子空间，直接"抹掉"后门参数的痕迹。

- 有干净参考模型时：直接对齐
- 无干净参考模型时：使用 EMA 轨迹或早期 checkpoint 作为参考

**Step 4：抗反弹训练（Anti-rebound Training）**

这是 BasinBreaker 最核心的创新。

传统防御只关心"当前 ASR 降了没有"，但持久性后门的特点是：防御后 ASR 暂时下降，但后续微调后 ASR 又反弹回来。

Anti-rebound 训练的做法是：**在防御过程中，显式模拟未来的干净微调**，把"微调 k 步后的 ASR"也写进优化目标。

$$\mathcal{L}_{\mathrm{anti\text{-}rebound}} = \mathcal{L}_{\mathrm{trigger}}(\theta + \Delta\theta_{\mathrm{sim}})$$

其中 $\Delta\theta_{\mathrm{sim}}$ 是模拟 k 步干净微调后的参数变化。

目标不仅是降低当前 ASR，还要降低未来可能反弹的 ASR。

### 4.3 创新点总结

| 创新点 | 说明 | 与现有工作的区别 |
|-------|------|---------------|
| 几何诊断视角 | 从损失景观几何角度理解后门持久性 | 现有防御不分析后门的参数空间几何 |
| 曲率感知子空间识别 | 用 Hessian/Fisher 信息定位后门子空间 | 现有方法多基于激活值或梯度大小 |
| 盆地破坏机制 | 主动提升后门区域曲率，破坏平坦盆地 | 现有方法只做"压低 ASR"，不改变景观形状 |
| 抗反弹目标 | 显式优化"未来微调后的 ASR" | 现有方法只优化"当前 ASR"，不考虑反弹 |

**一句话定位**：BasinBreaker 是第一个 persistence-aware 的多模态后门防御方法——它不只是清除后门，而是破坏后门赖以存活的参数空间结构。

## 五、前期验证实验规划

### 5.1 实验总体目标

在正式开展主实验之前，我们需要通过一系列前期实验验证三个核心假设：

1. **持久性后门确实存在反弹现象**（不只是 BadCLIP++ 的特例）
2. **后门区域确实对应低曲率/平坦盆地**（几何假设成立）
3. **破坏盆地确实能降低反弹**（防御思路可行）

### 5.2 实验路线图

**P0：环境搭建与 BadCLIP 复现**
- 基于 BadCLIP 开源代码搭建实验环境
- 复现 BadCLIP 在 CLIP ViT-B/32 上的攻击效果
- 成功标准：ASR > 95%，CA 下降 < 2%

**P1：验证反弹现象**
- 对 BadCLIP 后门模型施加多种防御（FT、CleanCLIP）
- 防御后继续干净微调 1/5/10/20/50 epochs
- 观察 ASR 是否出现反弹
- 成功标准：至少一种防御后出现 ASR 反弹 > 10%

**P2：Hessian 特征谱分析**
- 计算后门模型和干净模型的 Hessian 最大特征值
- 对比后门方向和干净方向的曲率差异
- 成功标准：后门方向的曲率显著低于干净方向

**P3：子空间可分离性验证**
- 计算 trigger-sensitive 方向和 clean-sensitive 方向
- 分析两组方向的夹角/重叠度
- 成功标准：两组方向可分离（夹角 > 60°）

**P4：最小化盆地破坏实验**
- 仅沿可疑方向做 sharpness ascent
- 观察 ASR 下降和反弹情况
- 成功标准：反弹 ASR 相比不做 sharpness ascent 降低 > 15%

**P5：跨攻击泛化验证**
- 对 Patch、Blended 等非持久性攻击重复 P1-P2
- 验证几何假设是否在不同攻击类型上成立

**P6-P8：SAM 持久性增强验证**
- P6：将 SAM 优化器集成到 BadCLIP 训练中，验证是否能增强持久性
- P7：对 SAM 增强后的后门做 Hessian 分析，验证平坦化效果
- P8：用 BasinBreaker 防御 SAM 增强后的后门，验证防御有效性

### 5.3 Go/No-Go 决策树

```
P1 结果：是否观察到反弹？
├── 是 → 继续 P2（验证几何假设）
│   ├── P2 成功（低曲率确认）→ 继续 P3-P4（验证防御思路）
│   │   ├── P4 成功 → 全力推进 BasinBreaker 主实验 ✅
│   │   └── P4 失败 → 调整防御策略，考虑替代方案
│   └── P2 失败（曲率假设不成立）→ 转向其他解释框架
└── 否 → 检查实验设置，尝试更强攻击或更长微调
    └── 仍无反弹 → 重新评估研究方向
```
